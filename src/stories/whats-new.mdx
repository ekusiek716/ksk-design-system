## 2. Checkbox — ポリモーフィック化

`label` prop を渡すと自動で行レイアウト（フィルタサイドバー型）に切り替わる。

👉 [Row Layout (label/description/count)](?path=/story/components-checkbox--row-layout)
👉 [Real UI — Filter Sidebar](?path=/story/components-checkbox--filter-sidebar)

```tsx
// 素のチェックボックス（従来通り）
<Checkbox checked={...} />

// 行レイアウト（新）
<Checkbox label="正社員" description="フルタイム" count={1234} />
```

---

## 3. Alert — prop ベース API ＋ 自動アイコン

`variant` に応じて `Warning2` / `InfoCircle` / `TickCircle` / `CloseCircle` を自動表示。
旧 `<Banner>` の用途を完全カバー。

👉 [Prop-based API (auto icon)](?path=/story/components-alert--prop-based-api)
👉 [Prop-based + action](?path=/story/components-alert--with-action)

```tsx
// 旧来の composable（そのまま動く）
<Alert variant="success">
  <AlertTitle>送信完了</AlertTitle>
  <AlertDescription>送信しました</AlertDescription>
</Alert>

// 新: prop ベース（アイコン自動）
<Alert variant="warning"
  title="メンテ予定"
  description="1/1 0:00〜3:00"
  action={<Button size="sm">詳細</Button>}
/>
```

## 6. AppHeader — 5 layout × @container SP/PC 切替

`layout` prop で 5 モードに（`default` / `logo` / `logo-center` / `back` / `back-search`）。
`nav` prop で PC 用 inline ナビ。`bottomSlot` で下段スロット。

👉 [AppHeader stories](?path=/story/patterns-appheader--default)

```tsx
<AppHeader
  layout="logo"
  logo={<Logo />}
  nav={[{label: "求人", href: "/jobs"}, {label: "企業", href: "/companies"}]}
  trailing={<UserMenu />}
/>
```

---

## 7. data-* 属性の徹底

CVA バリアントを持つコンポーネントすべてに `data-slot` / `data-variant` / `data-size` / `data-side` / `data-state` を出力。
**E2E / CSS フックは className ではなく data-* で**。

対象: Button / Badge / Card / Alert / Chip / Sheet / Toast / SyncStatusBadge /
CoachMark / DropdownFilter / Calendar / ProductCard (card-link)
