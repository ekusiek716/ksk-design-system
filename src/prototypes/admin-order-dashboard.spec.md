# 受注管理 — 仕様

Notion の PRD を貼り付ける運用のサンプル。`src/prototypes/<slug>.spec.md` を置くと
詳細画面の右パネルに表示される（依存追加なしの軽量 Markdown でレンダリング）。

## 目的

EC バックオフィス担当者が、当日の受注状況を一目で把握し、
**未対応の注文をすばやく捌く**ための管理画面。

## 主要要件

- KPI カードで「本日の売上 / 注文数 / 発送待ち」を常時表示する
- ステータスタブ（すべて / 新規 / 発送待ち / 完了）で注文を絞り込める
- 注文テーブルは**複数選択**でき、選択行に対して一括操作を行える
- 各行から個別アクション（詳細・編集・キャンセル）を呼び出せる

## 画面構成

1. ヘッダー（検索 + 担当者メニュー）
2. KPI カード行
3. ステータスタブ
4. 注文テーブル（チェックボックス / 一括操作バー）

## 補足

- 一括操作は「発送済みにする」「CSV 出力」を想定。
- 詳細仕様・受け入れ条件は元の PRD（上部の `PRD` リンク）を参照。
- 強調したい語は **太字**、識別子は `inline code` で表記できる。
